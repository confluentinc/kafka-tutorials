1. Get Confluent Platform

If you haven't already, get Confluent Platform.

{% include shared-content/docker-install.txt %}

2. Initialize the project

To get started, make a new directory anywhere you'd like for this project:

mkdir transforming-events && cd transforming-events

Then create the following Gradle build file, named build.gradle for the project:

{% include_raw recipes/transforming/kafka/code/build.gradle %}

Next, create a directory for configuration data:

mkdir configuration

Then create a development file at configuration/dev.properties:

{% include_raw recipes/transforming/kafka/code/configuration/dev.properties %}

3. Create schemas for the events

Create a directory for the schemas that represent the events in the stream:

mkdir -p src/main/avro

Then create the following Avro schema file at src/main/avro/input-movie-event.avsc for the publication events:

{% include_raw recipes/transforming/kafka/code/src/main/avro/input-movie-event.avsc %}

Then create the next one src/main/avro/parsed-movies.avsc for the publication events:

{% include_raw recipes/transforming/kafka/code/src/main/avro/parsed-movies.avsc %}

Because these Avro schemas are used in the Java code, it needs to compile it. Run the following:

gradlew build

5. Compile and run the Kafka program

In your terminal, run:

{% include_raw recipes/transforming/kafka/harness/recipe-steps/dev/build-uberjar.sh %}

Now that an uberjar for the Kafka application has been built, you can launch it locally. When you run the following, the prompt won't return, because the application will run until you exit it:

{% include_raw recipes/transforming/kafka/harness/recipe-steps/dev/run-dev-app.sh %}

6. Produce events to the input topic

In a new terminal, run:

{% include_raw recipes/transforming/kafka/harness/recipe-steps/dev/console-producer.sh %}

When the console producer starts, it will log some messages and hang, waiting for your input. Type in one line at a time and press enter to send it. Each line represents an event. To send all of the events below, paste the following into the prompt and press enter:

{% include_raw recipes/transforming/kafka/harness/recipe-steps/dev/input-events.json %}

7. Consume transformed events from the output topic

Leaving your original terminal running, open another to consume the events that have been transformed by your application:

{% include_raw recipes/transforming/kafka/harness/recipe-steps/dev/console-consumer.sh %}

After the consumer starts, you should see the following messages. The prompt will hang, waiting for more events to arrive. To continue studying the example, send more events through the input terminal prompt. Otherwise, you can Control-C to exit the process.

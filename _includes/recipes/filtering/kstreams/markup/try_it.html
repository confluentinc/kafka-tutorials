1. Get Confluent Platform

If you haven't already, get Confluent Platform.

2. Initialize the project

To get started, make a new directory for this project:

mkdir filter-events && cd filter-events

Then create the following Gradle build file, named build.gradle for the project:

{% include_raw recipes/filtering/kstreams/code/build.gradle %}

Next, create a directory for development environment configuration data:

mkdir -p src/main/resources/dev

Then create a development file at src/main/resources/dev/config.properties:

{% include_raw recipes/filtering/kstreams/code/src/main/resources/dev/config.properties %}

3. Create a schema for the events

Create a directory for the schemas that represent the events of the stream:

mkdir -p src/main/avro

Then create the following Avro schema file at src/main/avro/user.avsc:

{% include_raw recipes/filtering/kstreams/code/src/main/avro/user.avsc %}

Because we will use this Avro schema in our Java code, we'll need to compile it. Run the following:

gradle build

4. Create the Kafka Streams topology

Create a directory for the Java files in this project:

mkdir -p src/main/java/io/confluent/developer

Then create the following file at src/main/java/io/confluent/developer/FilterEvents.java:

{% include_raw recipes/filtering/kstreams/code/src/main/java/io/confluent/developer/FilterEvents.java %}

5. Compile and run the Kafka Streams program

In your terminal, run:

{% include_raw recipes/filtering/kstreams/code/recipe-steps/build-uberjar.sh %}

Now that an uberjar for the Kafka Streams application has been built, you can launch it locally by running:

{% include_raw recipes/filtering/kstreams/code/recipe-steps/run-dev-app.sh %}

6. Produce events to the input topic

In your terminal, run:

{% include_raw recipes/filtering/kstreams/code/recipe-steps/console-producer.sh %}

When the console producer starts, you'll have a prompt to enter in one line at a time. Each line represents an event. Paste the following into the prompt and press enter:

{% include_raw recipes/filtering/kstreams/code/recipe-steps/input-events.json %}

7. Consume filtered events on the output topic

Leaving your original terminal running, open another to consume the events that have been filtered by your application:

{% include_raw recipes/filtering/kstreams/code/recipe-steps/console-consumer.sh %}
